# 同题多路线评估

## 评估方法

本轮引入 Meta Skill 的评估口径，但不把 Meta Skill 变成 ReaderLab 阅读输出仿型。采用的约束是：

- 不奖励字段存在，必须看输出是否改善最终用户结果。
- 组合路线必须优于 baseline，并解释失败断言。
- 参考扫描要收窄，只吸收高价值模式，明确不吸收什么。
- 进入生产前必须有 gate，不能把 worker PASS 当完成。

## 能力边界

| 路线 | 擅长 | 不擅长 | 是否可独立作为主链 |
|---|---|---|---|
| Cangjie-only | 候选生成、候选筛选、通过/降级/拒绝 | 读者体验、来源约束、阅读重要性判断 | 否 |
| Li Jigang-only | 材料定性、缺口判断、机制传导、读者认知转变 | 工程化候选池、来源复核、反过度重组 | 否 |
| ReaderLab-current-chain | 把现有字段串成手工链路 | 新增判断力、fresh source-grounded、盲评证据 | 否 |
| Combo-A | 先定性再筛选，能减少口号候选 | source-grounded 后置，可能事后补证 | 暂不推荐 |
| Combo-B | 来源先约束，再筛选，再表达 | 仍需 fresh parse、blind A/B、迁移测试 | 候选主链 |

## 不可抽离环节

- 仓颉筛选不能脱离主问题定义和来源边界。否则会筛出很像方法论、但不一定是原书主线的内容。
- 李继刚表达不能脱离候选筛选和反升格边界。否则会把文本写得更顺，却离原文更远。
- source-grounded 不能脱离 fresh parse、location refs 和人工 spot-check。继承旧 refs 只能算线索，不能算证据。
- ReaderLab 当前链路不能脱离人工判断。它证明的是手工闭环方向，不是生成器能力。

## 参考扫描是否过宽

评估阶段扫描五条路线是合理的，因为本轮目的就是 bakeoff。但如果进入生产模板，这个扫描已经偏宽。

下一步不应吸收：

- 李继刚的文风。
- 仓颉的 Skill 化倾向。
- ReaderLab 当前字段形态。
- 三套方法并排贴成大表。
- 乔木和 book-to-skill 的额外加分项。

下一步只吸收三个职责：

1. 来源硬约束。
2. 候选筛选。
3. 受控机制表达。

## 是否只奖励字段存在

ReaderLab-current-chain 最像字段齐全：主问题、候选池、部件地图、亮点边界都有，但主要是复现已有手工链路，新增判断力不足。

Cangjie-only 和 Li Jigang-only 不是单纯填字段。前者确实提升“选不选”，后者确实提升“为什么”。但它们各自不能独立处理 ReaderLab 的完整结构化提炼风险。

Combo-B 暂时不是字段拼接，因为它的三步有阻断关系：

- 来源约束先阻断不可说内容。
- 候选筛选再阻断不可升格内容。
- 机制表达只能处理已经通过或降级的判断。

这个判断仍需 blind A/B 证明。

## Output Risk

对 ReaderLab 最终阅读包的主要风险：

- 把“算法之道”的顺序讲错。`v101-16.xhtml` 显示顺序本身就是核心。
- 把“睡工厂”升格为常态原则。`v101-14.xhtml` 已给紧急边界。
- 把“工厂即产品”和“第一性原理成本拆解”讲成口号，而不是制造系统和成本结构判断。
- 把多行星、AI、人口、监管等宏大叙事直接升格，忽略外部核验。
- 把 69 法则整体升格，而不是当混合索引。
- 输出变成报告字段堆叠，读者看到很多判断名词，但不知道原文在哪里、为什么重要、哪里不能用。

## 应采用的 Gate

1. Fresh source-grounded gate：每个通过候选必须有新抽取的 EPUB 位置、原文范围、可说/不可说边界。
2. Candidate decision gate：每个候选必须给出通过、降级或拒绝理由，不能只列标题。
3. Anti-overreach gate：专门检查宏大叙事、拼命工作、使命免检、混合索引整体升格。
4. Detachability gate：说明每个仿型环节能否独立抽取，依赖哪些上下游。
5. Blind A/B gate：Combo-B 必须和 Li-only、Cangjie-only、ReaderLab-current-chain 盲评对比，不能自称组合增益。
6. Reader output gate：检查最终读者页是否更清楚，而不是只让审计报告更完整。
7. Migration gate：换第二类材料小测，证明不是《埃隆之书》特化。

## 下一步最高价值改进

先做一个轻量 Combo-B 验证模板：

材料范围 -> 来源断言 -> 候选筛选 -> 受控机制表达 -> 拒绝/降级清单 -> 读者页风险检查。

模板只拿 5-8 个候选跑完整 gate，并与 Li-only 做 blind A/B。不要扩写《埃隆之书》阅读内容，不要把模板包装成正式 Skill，不要直接改生成器。

## 主 Agent 最终判断

本轮能回答用户提出的组合问题：单抽某一环不天然可靠，组合也不天然更好。当前证据显示 Combo-B 比单一路线更符合 ReaderLab 的风险结构，但只到“候选主链”级别。

下一步若 Combo-B 在 blind A/B 或第二类材料迁移中不胜出，就必须降级为实验路线，选择最强单一路线或局部 gate 继续推进。
