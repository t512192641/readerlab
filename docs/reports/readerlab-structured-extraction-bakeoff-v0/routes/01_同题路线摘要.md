# 同题路线摘要

## 共同任务

五条路线处理同一材料和同一输出契约：

- 材料定性。
- 主问题。
- 全书主线。
- 部件 II 结构地图。
- 候选池：至少 8 个候选，并标通过、降级或拒绝。
- 3-5 个亮点提炼，并说明入选理由、相邻内容不升格理由、来源范围和边界。
- 方法自评。

## Cangjie-only

主能力：候选生成和筛选。

它把《埃隆之书》定性为方法与案例汇编，而非传统传记。主问题是：如何选择足够重要的问题，并用物理求真、执行系统、组织反馈和制造系统把看似不可能或太慢的技术任务压成现实产品。

主要通过项：

- 第一性原理成本拆解。
- 算法之道。
- 工厂即产品 / 制造即护城河。
- 文明风险优先级，但必须强边界。

主要降级或拒绝：

- 直接沟通、极度紧迫感、目标选择降级。
- 69 法则整体、单纯拼命工作、共情表达、多行星叙事整体拒绝。

主 Agent 审查：仓颉路线确实提升“选不选”的判断，尤其能把候选拆成通过、降级、拒绝。但它不适合独立成为 ReaderLab 主链，因为 reader output 和 source-grounded 都弱，且容易把“可转成方法”误当成“阅读上最重要”。

## Li Jigang-only

主能力：深读定性和机制传导。

它把主问题定为：宏大目标如何被压成工程系统、组织系统、制造系统和公司行动，同时避免天才崇拜、拼命工作、使命免检。

主要通过项：

- 第一性原理成本拆解。
- 算法之道。
- 工厂即产品。
- 组织边界会变成产品缺陷。

主要降级或拒绝：

- 目标函数、前线领导 / 睡工厂、速度、直接沟通降级。
- 69 法则整体、单纯拼命工作、企业即慈善、多行星整体拒绝直接升格。

主 Agent 审查：李继刚路线最强在“为什么重要”和“机制如何传导”，比普通摘要更能说明缺口、增量和不增量。但它不能独立进生产链，因为候选筛选和来源复核不够硬，强解释力可能把解释者重组写得像原书主张。

## ReaderLab-current-chain

主能力：复现当前手工链路。

它已经把主问题、候选池、通过/降级/拒绝、部件地图和亮点边界放进一条判断链。主线是：目标选择 -> 第一性原理 -> 工程 -> 组织系统 -> 删除/简化/加速/自动化 -> 制造系统 -> 公司案例 -> 文明风险边界。

主 Agent 审查：这条路线是有价值的 baseline，但不能奖励为胜出。它字段齐、结构齐，但主要继承已有手工链路和 location refs，新增判断力不足，尤其没有 fresh source-grounded contract 和 blind A/B。

## Combo-A

顺序：李继刚定性 -> 仓颉候选筛选 -> source-grounded 后置边界。

组合增益：

- 李继刚先收窄主问题，避免候选池被口号、年表、名人故事稀释。
- 仓颉把高层机制拆成可筛选候选，并保留通过、降级、拒绝。
- source-grounded 后置裁掉越界解释和伪亮点。

主 Agent 审查：Combo-A 有真实增益，但顺序风险偏高。source-grounded 后置时，如果前面已经形成强解释，后置审计容易变成补证，而不是阻断。它适合作为备选，不推荐作为主链。

## Combo-B

顺序：source-grounded 前置 -> 仓颉候选筛选 -> 李继刚受控表达。

组合增益：

- source-grounded 前置先决定“什么能说”，减少把选编材料当一手事实、把外部争议命题当已证明结论的风险。
- 仓颉再决定“什么值得升格”，保留真实取舍。
- 李继刚最后只表达已通过或降级的判断，把候选组织成机制传导，而不能反向改写候选和来源边界。

主 Agent 审查：Combo-B 当前暂时胜出。它不是因为字段更多，而是因为三个环节互相约束：来源约束阻断候选，候选筛选阻断表达，表达只能改善读者理解。它仍缺完整 fresh EPUB contract、blind A/B 和第二类材料迁移测试，因此只能进入下一轮验证。

## 主 Agent 排序

1. Combo-B：下一轮候选主链。
2. Combo-A：备选。
3. Li Jigang-only：最强单一路线，但只能作受控表达层。
4. Cangjie-only：中段筛选引擎。
5. ReaderLab-current-chain：baseline 和契约雏形。
