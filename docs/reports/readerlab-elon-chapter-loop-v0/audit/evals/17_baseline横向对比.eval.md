## 结论

ReaderLab 目前仍是五者里最适合作为读者第一入口的全书总结。它的优势是主线清楚、自然可读、边界锋利，能把 15 个章节合成“让现实先赢，然后把愿景做成系统”的读者向理解。

但四个 baseline 各自补上了 ReaderLab 不能直接承担的层：

- Cangjie 最强在“可迁移方法单元的验证链”。它保留 framework / principle / case / counter-example / glossary 候选池，再用 V1/V2/V3 判断通过、降级或拒绝。它提醒 ReaderLab 后续方法论不能只从顺滑叙事里抽原则，必须说明候选从哪里来、为什么通过、为什么降级。
- 李继刚最强在“深读批评”。它把书的材料属性、增量 / 不增量、机制链、认知旅程、可复核层级拆得最清楚。它对 ReaderLab 的有效批评是：只给结构图还不够，必须回答这本书补了什么缺口、机制如何传导、哪些结论只是解释者重组、哪些需要外部复核。
- book-to-skill 最强在“能力边界”。它把整书不能 skill 化说清楚，只保留第一性原理可行性拆解、算法之道流程瘦身、最短沟通路径与组织边界诊断、交付系统瓶颈定位、单一核心指标反馈回路五个能力。它也明确拒绝极限硬核工作、多行星叙事、AI 生存风险和 69 法则整体直接变成 Skill。
- 乔木最强在“陪读现场”。它不追求全书总论，而是设计章节共读问题、正文旁批注触发点、误读纠偏和读者行动。它提醒 ReaderLab 的阅读包不能只有高阶总结，还要在读者真正会误读、会被打动、会想迁移的地方给出提问入口。

## 排名不是同一维度

如果目标是“读者先读哪一篇”，ReaderLab 第一。

如果目标是“提炼 ReaderLab 方法论”，不能只看 ReaderLab 自己总结，应该组合：

1. ReaderLab 的 reader-facing 叙事能力。
2. Cangjie 的候选池和三重验证。
3. 李继刚的深读缺口 / 增量 / 机制 / 可复核分层。
4. book-to-skill 的触发、输入、步骤、输出、边界筛选。
5. 乔木的正文旁共读问题、批注触发和读者行动设计。

## 对 ReaderLab 的方法缺口

ReaderLab 自己总结已经解决了“读者如何重新理解这本书”的问题，但还没有解决“ReaderLab 方法如何稳定复用”的问题。baseline 对比暴露出后续草案至少要补五层：

1. 候选发现层：不能只输出顺滑总结，要保留候选方法、案例、反例、术语和被拒绝项。
2. 验证层：每个可迁移方法要说明书内多语境支撑、书外预测力和独特性。
3. 材料审计层：区分书内直接支撑、解释者重组和外部待验证结论。
4. 能力边界层：把“阅读洞察”与“可触发 Skill”分开，拒绝把高风险理念直接工具化。
5. 陪读交互层：给读者在正文附近可批注、可追问、可迁移的小问题，而不是只给全局结论。
