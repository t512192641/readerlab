## 为什么旧版不够

旧 LifeAtlas `06_能力地图.md` 有解释性价值，但仍容易停留在文章式总览：读者能看到大结构，却不能从合同层确认 24 个 Skill 是否全覆盖、每个能力域是否有可回链证据、跨 Skill 路由是否结构化、辅助资产是否已审计。

旧 repo demo 更明确是 sample-level：只登记 7 个代表位置，DBS-A02 到 DBS-A05 只能 partial。本 v1 的修正是：

- `coverage_plan.known_total_units=24`
- `covered_units` 列出全部 24 个 Skill 名
- 每个 Skill 在 `source-registry` 和 `location-map` 中至少有一个可回链位置
- 能力地图按任务族组织，不按目录清单组织
- 读者稿和审计稿分离

## 审计结论

v1 能证明 ReaderLab 在 `dbs-suite` 样张上已经从“代表性目录 demo”推进到“24 Skill full inventory 的能力地图样张”。但它还不能证明：

- ReaderLab 生成器能自动稳定产出同等质量。
- `dbs-suite` 本身已经安装、启用或执行通过。
- `dbs-content-system` 工具链和模板已经可运行验收。
- 人工读者已经接受这张能力地图。
