# 阅读入口

这套阅读包把《埃隆之书》先当成一组可追溯的思想材料，而不是直接当成传记、励志书或管理学手册。

## 你先抓住三件事

1. 这本书的主链条是：目标 -> 第一性原理 -> 工程价值 -> 组织执行 -> 制造系统 -> 公司案例 -> 文明风险。
2. 最容易误读的是第二部分。它看上去像“极致努力”，实际更值得读的是短路径沟通、算法之道、速度和工厂即产品。
3. 第三、四部分不是单纯故事和愿景。Zip2、PayPal、Tesla、SpaceX 是案例压力测试；多行星、AI、风险是边界压力测试。

## 建议读法

先读 `01_全书阅读路线.md`，知道每部分解决什么问题；再读 `02_全书地图.md`，建立整本书的机制链；然后读 `03_深读卡组.md`，把第二部分拆成可批注的卡；最后用 `04_案例_反例_术语.md` 做复盘。

批注时，不要只写“赞同/不赞同”。更有用的问题是：这个方法在我的场景里能不能迁移？它的反例是什么？它需要哪些安全边界？
