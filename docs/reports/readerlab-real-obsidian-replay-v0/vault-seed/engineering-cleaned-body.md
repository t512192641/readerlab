# ReaderLab UI Replay Test - Engineering Cleaned Body

这是一份用于真实 Obsidian 批注回放的工程材料净化正文。它模拟 Skill、工作流协议或工程说明被转成可读材料时的主体内容。

净化正文必须保留用途、触发条件、用户意图和核心流程。ReaderLab 可以剥离安装命令、重复模板、路径噪音和宿主环境细节，但不能删掉读者理解这个工作流所需的约束、失败条件和输出要求。

被剥离的执行外壳进入 audit 或附录。附录用于读者可能需要回看的执行细节，audit 用于证明来源、位置和判断链路。二者都不能冒充材料主体，也不能污染主阅读页。

测试锚点：

- E1: 净化正文必须保留用途、触发条件、用户意图和核心流程
- E2: 被剥离的执行外壳进入 audit 或附录
