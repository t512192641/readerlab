# High-Order Explanation Method

> Authority boundary: 本文件是高阶讲解方法说明，不是当前任务源。当前要处理哪一章、是否进入全书总结或 final boss，以 `docs/current-task.md` 为准。
>
> 2026-07-01 GPT Pro review 后补充：`high-order-explanation.v1` 只证明 reader-facing 高阶讲解层，不证明完整 ReaderLab 阅读包，也不证明可迁移方法核。完整方法核还必须经过 Body Track Gate、Material Profile、Claim Ledger、Candidate Tournament、Skillization Gate 和 Annotation Trigger。

## 目的

本方法定义 ReaderLab 的“认知升级讲解”如何生产。它解决的问题不是“如何把章节总结得更顺”，而是让 AI 陪读稳定产生非摘要型认知增量：把正文放回更大的问题，解释正文内部机制，选择真正照亮正文的高维镜头，并给出可吸收、需降级、必须拒绝的判断。

本方法是 Agent 内部生产协议，不是 reader-facing 栏目。读者最终看到的仍然是一段或几段自然讲解。

## 核心原则

高阶讲解必须同时满足两件事：

1. 讲清楚材料本身的重点、要点和结构。
2. 让读者获得“原来这章可以这样看”的认知增量。

它可以引入跨学科思维模型、历史观、商业案例、组织管理、工程系统、心理机制和用户既往经历中的相似结构，但这些都只是镜头。镜头必须反向照亮正文，不能抢走正文。

## 内部生产协议

```text
正文锚定
-> 普通读法识别
-> 升维问题
-> 机制建模
-> 镜头选择
-> 吸收 / 降级 / 拒绝
-> 自然讲解
-> 认知增量验收
```

### Step 1. Source Anchor

输入正文，抽取 8-15 个正文锚点。

每个锚点只记录：

- 正文说了什么。
- 支撑哪个判断。
- 不能支持什么判断。

禁止在本步写宏大解释。锚点的作用是约束后续讲解，防止 AI 把外部模型写成原文结论。

### Step 2. Baseline Summary Trap

写出普通总结会怎么讲。这个结果不进入读者页，只用于识别要超越的浅层读法。

如果最终稿只比 baseline summary 更流畅、更漂亮，但没有新增问题框架、机制链、镜头、边界或裁决，本轮判 FAIL。

### Step 3. Upgrade Question

把章节或模块主题改写成更大的问题。

通用公式：

```text
表面主题：这一章在讲 X。
升维问题：在什么约束下，谁要把什么资源转化成什么结果，同时避免什么失败？
```

对书籍 / 长文：

```text
这一章不只是讲 X，而是在问：
当 A 约束存在时，人 / 组织 / 系统如何通过 B 机制得到 C 结果，同时避免 D 风险？
```

对 Skill / 工程材料：

```text
这个 Skill 不只是执行 X，而是在问：
当用户意图不完整、输入不稳定、工具有边界、输出要可复用时，
它如何通过流程、约束、确认点和输出契约把任务变得可控？
```

### Step 4. Mechanism Graph

把正文里的观点、案例、流程和判断串成机制链，而不是罗列亮点。

最低字段：

```text
input：材料中的关键资源或前提
constraint：它面对的限制
action：材料主张采取的动作
feedback：现实如何返回信号
output：它想得到的结果
failure_mode：它最容易坏在哪里
boundary：它在哪些条件下不能迁移
```

没有机制链的讲解容易退化为普通总结。机制链也必须受正文锚点约束，不能靠外部常识强行补全。

### Step 5. Lens Auction

镜头选择不是套模型，而是给正文找最能照亮它的视角。

每个候选镜头必须回答：

- 正文哪里触发它？
- 它解释了什么机制？
- 它揭示了什么普通总结看不到的边界或风险？
- 如果不用它，读者会少理解什么？
- 它的发散风险是什么？

照亮测试：

```text
一个镜头只有同时满足三点，才能 Promote：
1. 它解释了正文里的核心机制。
2. 它揭示了普通总结看不到的边界或风险。
3. 它能反向帮助读者读懂正文，而不是让读者注意力转向模型本身。
```

评分规则：

```text
正文触发强度：0-3
解释力：0-3
边界揭示力：0-2
读者认知增量：0-2
发散风险：0 到 -3

总分 >= 7，且发散风险不低于 -1，才能进入 reader-facing。
每篇高阶讲解最多 2 个主镜头 + 1 个边界镜头。
没有镜头通过时，只讲材料内部机制。
```

常见镜头和触发条件：

- 系统论 / 控制论：反馈、信号、目标、误差校正、瓶颈、局部最优、工具、人机协作、信息流。
- 组织理论：招募、角色、权责、沟通路径、文化、层级、绩效、人才密度。
- 经济学 / 激励：稀缺、回报、机会成本、风险承担、资源配置、激励相容。
- 心理学：恐惧、动机、身份认同、被喜欢、反馈回避、情绪防御。
- 工程 / 产品：原型、验证、制造、迭代、约束、可靠性、输出契约。
- 历史视角：时代结构、制度演化、组织形态变化、长期周期；不默认使用。
- 伦理 / 权力边界：强控制、高压、牺牲、情绪贬低、人格判断、文明叙事正当化。

### Step 6. Judgment Gate

对候选观点做 Promote / Keep / Downgrade / Reject。读者页不展示审计表，但自然讲解里必须完成判断。

吸收 Promote：

- 正文有明确支撑。
- 不是孤立金句，而是机制的一部分。
- 能跨场景迁移。
- 有清楚前提。
- 误用风险可控。
- 能变成读者的判断框架或行动方法。

降级 Downgrade：

- 正文有启发，但上下文依赖强。
- 只适合危机、特殊组织、特殊能力结构或特殊工具边界。
- 有外部证据缺口。
- 可作为提醒、案例、边界，但不能当原则。
- 容易被普通读者过度泛化。

拒绝 Reject：

- 只是口号或人格姿态。
- 冲击力强但机制不足。
- 复制后大概率造成伤害。
- 把个体极端条件当成普遍方法。
- 把 AI 推断或作者自述冒充事实规律。
- 绕过法律、安全、伦理、专业判断或人的基本边界。

### Step 7. Natural Explanation

把内部结果写成自然讲解。禁止展示内部字段，不写栏目墙。

建议形态：

```text
## 本章真正讲的是：自然标题

第一段：表面主题 + 真正大问题。
第二段：把正文里的关键内容串成机制。
第三段：引入 1 个最合适的高维镜头，解释它为什么照亮正文。
第四段：讲清边界，完成吸收 / 降级 / 拒绝。
第五段：收束为读者可带走的判断。
```

标题可以自然，不使用内部术语。例如：

- `## 本章真正讲的是：组织怎样校正现实`
- `## 读这章，不要只看“招天才”`

### Step 8. Delta Eval

和 baseline summary 对照。必须证明新版不是普通总结。

最低问题：

1. 是否把章节主题升级成了一个更大的问题？
2. 是否给出了正文内部的机制链，而不是只列重点？
3. 是否明确说出普通读者最容易读浅在哪里？
4. 是否至少有一个镜头真正照亮正文？
5. 外部模型 / 类比是否能反向解释正文，而不是并列摆放？
6. 是否明确区分原文支持、AI 解释、外部类比和待验证判断？
7. 是否完成吸收 / 降级 / 拒绝，而不是只说“有启发”？
8. 和 baseline summary 相比，是否让读者产生“原来这章可以这样看”的认知增量？

## 失败条件

出现以下任一情况，本轮高阶讲解失败：

- 只复述本章讲了什么。
- 只把旧 demo 改写得更流畅。
- 堆模型、名人、历史例子，但不能反向照亮正文。
- 没有升维问题。
- 没有机制链。
- 没有吸收 / 降级 / 拒绝。
- 把 AI 解释写成原文结论。
- 让高阶讲解抢掉正文主体。

- 新版 5 项中至少赢 4 项。

- 外部模型抢走正文注意力，FAIL。
- 仍像普通总结，FAIL。
